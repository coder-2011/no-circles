# TODO.md Planning Prompt

When planning a feature, convert work into a small markdown task inventory in root `TODO.md`.

Process:
1. Break the feature into 3-7 implementation tasks only when decomposition adds clarity.
2. Place tasks under repo-relevant headings only when that improves scanability.
3. Write each item as an actionable checkbox entry.
4. Keep the ordering practical so the next step is obvious.

Task quality bar:
- names the subsystem, file area, or user-visible outcome
- can be verified by a command, test, or visible behavior change
- is small enough to complete without hidden subprojects
